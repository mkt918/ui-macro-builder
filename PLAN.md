# UI Macro Builder — 開発計画

## プロジェクト概要

**授業用ビジュアルプログラミングツール**。
複数生徒が同時に使い、アルゴリズム（ループ・条件分岐・配列）を **ブロックの組立 → VBA コード自動生成 → 仮想 Excel でビジュアル確認** という流れで学べる。

### ターゲット
- **対象**: 高校生などの非エンジニア（Excel 基本操作はできるが、コード記憶ゼロ）
- **学習内容**: ループ処理・条件分岐・配列の基礎
- **授業スタイル**: 対面・複数生徒同時・生徒が自操作・先生は見回り

---

## ゴール

| # | ゴール |
|---|--------|
| 1 | ブロックをドラッグ＆ドロップして Excel マクロロジックを組み立てられる |
| 2 | 組み立てたブロックから VBA コードを自動生成できる |
| 3 | 仮想 Excel でマクロの **途中経過・ハイライト・アニメーション** をリアルタイム表示 |
| 4 | 難易度付き課題セット（★～★★★★★）を、解説・3段階ヒント・答え付きで提供 |
| 5 | 生成コードをワンクリックでコピー・Excel 貼り付け実行できる |

### 主要機能
- **課題セレクタ**: 難易度（★マーク）で一覧表示、タップで切り替え
- **ヒント段階表示**: ボタン押下で「ヒント1」→「ヒント2」→「ヒント3」を順番に表示
- **Excel プレビュー**: ステップごとの途中経過・セルハイライト・実行アニメーション
- **ブロック構成の永続化**: 課題ごとにセーブスロット（課題を切り替えても復元）
- **エラー警告**: エディタ内に赤く「接続できません」と表示

## ブロックカテゴリ（実装予定）

### フェーズ 1（MVP）—— 学習コア（ループ・条件分岐・配列）
- **セル操作** 
  - 📝 セルに値を入力
  - 🔍 セルの値を取得
  - 📋 セルをコピー
  - 🗑 セルをクリア
  
- **繰り返し**
  - 🔁 N 回繰り返す
  - 📊 範囲をループ
  
- **条件分岐**
  - ❓ もし〜なら
  - ↔ そうでなければ
  
- **書式設定**
  - 🎨 背景色を変える
  - 𝗕 太字にする
  - 🔠 フォントサイズ変更

### フェーズ 2
- **シート操作**: シートを追加、シートを切り替え
- **配列操作**: 配列を作る、配列から値を取得、配列に追加
- **計算**: 足す、引く、掛ける、割る

### フェーズ 3
- **関数呼び出し**: SUM、AVERAGE、COUNT など
- **ユーザー入力**: InputBox、MsgBox
- **エラー処理**: On Error ブロック

## 技術スタック（確定）

| レイヤー | 技術 | 理由 |
|----------|------|------|
| フロントエンド | Vanilla JS + HTML/CSS | シンプル・依存最小 |
| ブロックエンジン | [Blockly](https://developers.google.com/blockly) | ブロック定義と VBA 生成が分離・拡張性高い |
| VBA 生成 | Blockly カスタムジェネレーター | Blockly の標準機能 |
| 仮想 Excel | HTML テーブル + Canvas | 動的なセル更新・アニメーション対応 |
| 課題データ | JSON 形式 | 拡張・管理しやすい |
| ホスティング | GitHub Pages | 無料・生徒がリンク 1 つで使える |
| 開発用 AI | Claude Opus | VBA 生成精度・複雑処理の安定性重視

## Phase 6 — バグ修正 & 品質向上（計画: 2026-06-11）

コード精読＋実機検証で見つけた問題と、授業運用に必要と判断した機能。
優先度: 🔴必須 / 🟡推奨 / ⚪あると良い

### 🐛 バグ（実機で確認済み）

| ID | 優先 | 内容 | 影響 | 対策 |
|----|------|------|------|------|
| B1 | 🔴 | **ステップ爆発の上限なし**。ネストループで 50×50=2500、1000×1000=100万ステップ生成。各ステップでモデル全体を deep-copy するため、大きい値でブラウザがフリーズ | 生徒が大きい数を入れるとハングする | ステップ総数に上限（例: 5000）。超えたら生成を打ち切り「多すぎます」表示 |
| B2 | 🔴 | **配列ビジュアライザが巨大インデックスで暴走**。`Math.max(...keys)` 番目まで DOM 生成。arr(1000) を入れると 1000 セル描画 | フリーズ・描画崩れ | 表示上限（例: 30）。超過分は「…」省略 |
| B3 | 🔴 | **グリッド範囲外（I列以降・13行目以降）への書き込みが見えない**。モデルには入るが DOM セルが無く、生徒には「何も起きない」 | 「動かない」と誤解・学習の妨げ | グリッドを動的拡張 or 範囲外書き込みを検知して警告表示 |
| B4 | 🔴 | **リロードで全作業が消える**。`savedBlocks`・`solved` がメモリ上のみ。生徒が誤ってリロード/ブラウザを閉じると組んだブロックも進捗も消失 | 授業中の事故で泣く | localStorage に永続化（ブロック構成・クリア状況） |
| B5 | ⚪ | `Blockly.Theme.defineTheme` を毎回同名で再定義（テーマ切替のたび）。重複登録で警告・非効率 | 軽微 | テーマは初期化時に1度だけ定義して使い回す |
| B6 | ⚪ | `buildQuestModal()` が二重呼び出し（init と bindControls） | 軽微 | 1回に整理 |
| B7 | ⚪ | `loadTask` 内 `workspace.clear()` の change イベントで、新タスクIDに一瞬「空ブロック」を保存。順序依存で脆い | 実害小 | 復元中は保存を抑制するフラグ |
| B8 | ⚪ | `{i}` をループ外で使うと i=0 → "A0" など無効アドレス、無警告 | 軽微 | ループ外 `{i}`/`カウンタi` を警告 |

### ✨ 追加機能（授業運用で必要と判断）

| ID | 優先 | 機能 | 理由 |
|----|------|------|------|
| F1 | 🔴 | **作業＆進捗の自動保存（localStorage）** | B4対策。授業をまたいで続きができる。クエスト一覧に ✓ クリア表示も永続化 |
| F2 | 🟡 | **「全部消す」ボタン（確認ダイアログ付き）** | ゴミ箱へD&Dは小さい子に難しい。やり直しを楽に |
| F3 | 🟡 | **あまり(Mod)演算ブロック** | 偶数判定が今は `i=2,4,6` の力技。`i ÷ 2 のあまり = 0` で本来の偶数判定を学べる（条件分岐学習の質が上がる） |
| F4 | 🟡 | **クリア後「次のクエストへ」ボタン** | 達成→次へ の流れをスムーズに。学習リズム |
| F5 | 🟡 | **クリア演出の強化**（🎉紙吹雪 or チェック音、クエスト一覧の✓同期） | 達成感・モチベーション。レイトン風の手応え |
| F6 | ⚪ | **VBA コードのシンタックスハイライト** | 今はプレーン。色が付くとコードが読みやすく、Excelに貼る前の理解が進む |
| F7 | ⚪ | **クエスト内に「お手本の完成イメージ」表示** | ゴール状態のExcelプレビュー画像/表で、何を作るか一目で分かる |
| F8 | ⚪ | **ブロック構成の共有（URLエンコード）** | 生徒同士・先生へ提出。壁打ちで出た案 |

### 実装順（推奨）
1. **第1弾（🔴 安定性・事故防止）**: B1, B2, B3, B4 + F1 — まず「壊れない・消えない」を担保
2. **第2弾（🟡 学習体験）**: F3(Mod), F2(全消し), F4(次へ), F5(演出)
3. **第3弾（⚪ 仕上げ）**: F6, F7, B5–B8, F8

## Phase 7 — 仮想Excelへの直接入力 (2026-06-11) ✅ 完了

生徒が仮想 Excel のセルに数値・文字を直接入力できるようにした。
「元データを用意してマクロで処理する」という実 Excel に近い使い方が可能に。

| 項目 | 内容 |
|------|------|
| 編集 | セルをダブルクリック → 入力 → Enter/フォーカス外で確定（Escで取消） |
| 数値判定 | 数字なら Number、それ以外は文字列として保存 |
| マクロ連携 | 入力した初期データを土台にブロックが実行される（buildSteps に初期データを渡す） |
| 表示 | アイドル時・リセット時は初期データを表示。実行中は編集不可 |
| 永続化 | 課題ごとに初期データを localStorage 保存（リロードしても残る・検証済） |
| 新クエスト | t06「入力した数を2倍にしよう」を追加（A1入力→B1=A1×2） |

検証: A1=50 入力 → マクロ B1=A1×2 → B1=100・クリア判定OK、
セル編集→確定→localStorage保存→リロード復元 をブラウザで確認済み。

## 決定済み事項

| 項目 | 決定 | 理由 |
|------|------|------|
| **配布形式** | Web アプリ（GitHub Pages） | 生徒がリンク 1 つで使える |
| **ブロック UI** | Blockly | ブロック定義と VBA 生成が分離・拡張性 |
| **ブロック接続ルール** | 実装後に判断（柔軟に） | 初期版は接続可能性チェック簡易版 |
| **答え表示** | コード（VBA）のみ | ブロック構成は見せずコード確認で学習 |
| **新規課題時** | ブロック真っ白 | 毎回ゼロから組み直す＝学習効果 |
| **課題追加方法** | Claude に「こういう課題を追加で」 | 先生が簡単に拡張・更新できる |
| **Excel プレビュー** | ステップ・ハイライト・アニメーション | 「何が起きているか」の理解促進 |
| **ブロック構成保存** | 課題ごとセーブスロット | 課題切替後も復元される |
| **エラー表示** | エディタ内に赤く警告 | 実装後に接続ルール詳細化 |
| **実行速度** | 生徒が自由に調整可能 | ゆっくり追うか、パッと見るか選択 |
| **開発用 AI** | Claude Opus | VBA 生成品質重視 |
